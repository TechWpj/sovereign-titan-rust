## Technical Writing Guidance

**Structure:**
- Start with a brief overview of what the document covers.
- Include a prerequisites section (required knowledge, tools, versions).
- Use numbered steps for procedures, bullets for lists.

**API Documentation:**
- Document each endpoint: method, path, parameters, request body, response format.
- Include working code examples in at least one language.
- Note authentication requirements and rate limits.
- Document error responses and their meanings.

**Code Examples:**
- Make examples copy-pasteable — include imports and setup.
- Use realistic but simple data.
- Show both the request and expected response.

**Troubleshooting:**
- Include a troubleshooting section with common errors.
- Format as: Symptom -> Cause -> Solution.
- Link to related documentation for deeper dives.

**Maintenance:**
- Mark version-specific information clearly.
- Use admonitions for warnings, notes, and tips.
- Keep language precise — "click" not "hit", "enter" not "put in."
