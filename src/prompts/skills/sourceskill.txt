# Source Skill — Academic & Curated Research Sources

## Overview
When handling research tasks, Titan has access to curated academic and news sources
beyond standard web search. Use the `academic_search` tool for scholarly queries and
`web_search` for general results. The system automatically detects topic domains and
routes to the most relevant sources.

## Primary Sources (16 curated, free access)

### Scholarly / Peer-Reviewed (with APIs — use academic_search)
1. **Semantic Scholar** — https://www.semanticscholar.org
   - AI-powered academic search, TLDR summaries, citation graphs
   - Best for: CS, AI/ML, NLP, general science
   - API: api.semanticscholar.org/graph/v1/paper/search

2. **arXiv** — https://arxiv.org
   - Open-access preprint server
   - Best for: CS, AI/ML, physics, math, quantitative biology
   - API: export.arxiv.org/api/query (Atom XML)

3. **PubMed** — https://pubmed.ncbi.nlm.nih.gov
   - NCBI biomedical literature database (36M+ citations)
   - Best for: medicine, biology, health, biomedical
   - API: NCBI E-utilities (esearch + esummary)

4. **DOAJ** — https://doaj.org
   - Directory of Open Access Journals (20K+ journals)
   - Best for: broad scholarly coverage, open access
   - API: doaj.org/api/search/articles

5. **Hacker News** — https://news.ycombinator.com
   - Tech community discussions, startup news
   - Best for: CS, AI, tech industry, startups
   - API: hn.algolia.com/api/v1/search

6. **PLOS ONE** — https://journals.plos.org
   - Open-access multidisciplinary science journal
   - Best for: biology, medicine, general science
   - API: api.plos.org/search

### Scholarly (no direct API — use site: restriction in web_search)

## Tool Usage Guide

### academic_search
Use for: finding scholarly papers, preprints, citations, expert analysis
```
ACTION: academic_search
PARAMS: {"query": "transformer architecture attention mechanism", "max_results": 3, "domain": "ai"}
```
The tool automatically detects the topic domain and queries the top 4 relevant APIs.

### web_search with site: restriction
For sources without APIs, use site: in the query:
```
ACTION: web_search
PARAMS: {"query": "site:ssrn.com impact of remote work on productivity"}
```

## Override Rules

1. **User-specified sites always win**: If the user says "search on reuters.com" or
   uses "site:example.com", honor that — skip automatic source routing.

2. **Music playback → YouTube**: "play jazz music" routes to YouTube, not academic
   sources. But "research the history of jazz" routes to academic sources.

3. **Research indicators trigger academic sources**: Words like "research", "analyze",
   "study", "paper", "peer-reviewed", "citation" activate academic routing.
